# 11. Container With Most Water

## Pattern

Two Pointers

## Problem

You are given an integer array `height` where each element represents the height of a vertical line.

Find two lines that, together with the x-axis, form a container that can hold the maximum amount of water.

Return the maximum amount of water the container can store.

---

## Constraints

```text
2 <= height.length <= 10^5
0 <= height[i] <= 10^4
```

---

# Approach 1: Brute Force

## Idea

Try every possible pair of lines.

For every pair:

* Width = distance between the two lines.
* Height = smaller of the two lines.
* Area = Width × Height.

Keep track of the maximum area.

---

## Algorithm

1. Pick the left line.
2. Pair it with every line on its right.
3. Compute:

   * Width = `r - l`
   * Height = `min(height[l], height[r])`
   * Area = Width × Height
4. Update the maximum area.
5. Return the maximum area.

---

## Code

```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        res = 0

        for l in range(len(height)):
            for r in range(l + 1, len(height)):
                area = (r - l) * min(height[l], height[r])
                res = max(res, area)

        return res
```

---

## Dry Run

```python
height = [1,8,6,2]
```

Possible containers:

| Left | Right | Width | Height | Area |
| ---- | ----: | ----: | -----: | ---: |
| 1    |     8 |     1 |      1 |    1 |
| 1    |     6 |     2 |      1 |    2 |
| 1    |     2 |     3 |      1 |    3 |
| 8    |     6 |     1 |      6 |    6 |
| 8    |     2 |     2 |      2 |    4 |
| 6    |     2 |     1 |      2 |    2 |

```text

```

# Why Improve?

The brute-force solution checks every pair of lines.

Many of these comparisons are unnecessary.

Can we eliminate some pairs without missing the optimal answer?

Yes.

---

# Approach 2: Two Pointers (Optimal)

## Key Observation

The area depends on:

```text
Area = Width × min(height[left], height[right])
```

At every step:

* Moving either pointer decreases the width.
* Therefore, to get a larger area, we need a chance to increase the limiting height.

The shorter line is the limiting factor.

So:

* If the left line is shorter, moving the right pointer cannot increase the height of the container because the left line still limits the water.
* The only hope is to move the left pointer to find a taller line.

Similarly:

* If the right line is shorter, move the right pointer.

This allows us to safely discard many pairs.

---

## Why Move the Smaller Height?

Suppose:

```text
Left Height  = 3
Right Height = 8
```

Current area:

```text
Width × 3
```

If we move the taller line:

```text
Width decreases

Smaller height is still 3
```

The area can only decrease or remain the same.

So moving the taller pointer can never produce a better result.

The only possibility of increasing the area is to move the shorter pointer and hope to find a taller line.

---

## Algorithm

1. Place one pointer at the beginning.
2. Place another pointer at the end.
3. Compute the current area.
4. Update the maximum area.
5. Move the pointer with the smaller height.
6. Repeat until the pointers meet.

---

## Code

```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        res = 0

        l, r = 0, len(height) - 1

        while l < r:
            area = (r - l) * min(height[l], height[r])
            res = max(res, area)

            if height[l] < height[r]:
                l += 1
            else:
                r -= 1

        return res
```

## Dry Run

```python
height = [1,8,6,2,5,4,8,3,7]
```

### Initial

```text
l = 0 (1)
r = 8 (7)

```

Since:

```text

```

Move left.

### Next

```text
l = 1 (8)
r = 8 (7)

Area = 7 × 7 = 49
```

Maximum:

```text

```

Since:

```text
8 > 7
```

Move right.

Continue until the pointers meet.

Final answer:

```text
49
```

---

## Why Does This Work?

At every step:

```text
Width decreases.
```

Therefore, the only way to obtain a larger area is to increase the limiting height.

The limiting height is always:

```text
min(height[left], height[right])
```

So we always move the shorter line.

This guarantees that every discarded pair cannot produce a better answer.

# Learnings

* Two pointers are useful when the answer depends on both ends of a sorted or indexed structure.
* Always identify what limits the current answer.
* Here, the limiting factor is the shorter line.
* Moving the taller pointer cannot improve the area because the width decreases while the limiting height remains unchanged.
