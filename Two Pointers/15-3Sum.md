# 15. 3Sum

## Pattern

Sorting + Two Pointers

## Problem

Given an integer array `nums`, return all the unique triplets:

```text
[a, b, c]
```

such that:

```text
a + b + c = 0
```

The solution set must not contain duplicate triplets.

---

## Constraints

```text
3 <= nums.length <= 3000
-10^5 <= nums[i] <= 10^5
```

---

# Approach 1: Brute Force

## Idea

Try every possible triplet.

For every combination of three numbers:

* Calculate their sum.
* If the sum is `0`, store the triplet.
* Since duplicates are possible, sort each triplet before inserting it into a set.

---

## Algorithm

1. Pick the first element.
2. Pick the second element after it.
3. Pick the third element after the second.
4. If the sum equals zero:

   * Sort the triplet.
   * Insert it into a set.
5. Convert the set into a list.

---

## Code

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        ans = set()
        n = len(nums)

        for i in range(n - 2):
            for j in range(i + 1, n - 1):
                for k in range(j + 1, n):
                    if nums[i] + nums[j] + nums[k] == 0:
                        ans.add(tuple(sorted([nums[i], nums[j], nums[k]])))

        return [list(x) for x in ans]
```

---

## Complexity Analysis

### Time Complexity

```text
Three nested loops

O(n³)
```

### Space Complexity

```text
O(n)
```

for storing unique triplets.

---

# Why Improve?

The brute-force solution checks every possible triplet.

Most of these combinations are unnecessary.

Can we reduce the problem?

Yes.

---

# Key Observation

If we **fix one number**, the remaining problem becomes:

```text
Find two numbers whose sum equals

-target
```

This is exactly **Two Sum II (Sorted Array)**.

Therefore,

```text
3Sum

↓

Fix one element

↓

Solve Two Sum II using Two Pointers
```

---

# Approach 2: Sorting + Two Pointers (Optimal)

## Idea

1. Sort the array.
2. Fix one element.
3. Use two pointers to search for the remaining two numbers.

Since the array is sorted:

* If the sum is too small, move the left pointer.
* If the sum is too large, move the right pointer.

This avoids checking every possible triplet.

---

## Why Sort?

Sorting gives two major advantages:

### 1. Two Pointer Technique

Because the array is sorted:

```text
Move left

↓

Sum increases
```

```text
Move right

↓

Sum decreases
```

---

### 2. Duplicate Removal

After sorting:

```text
[-4,-1,-1,0,1,2]
```

Duplicate values become adjacent.

Therefore, they can be skipped easily.

---

## Algorithm

1. Sort the array.
2. Iterate through every element.
3. Skip duplicate first elements.
4. Initialize:

   * `left = i + 1`
   * `right = n - 1`
5. Compute:

```text
nums[i] + nums[left] + nums[right]
```

6. If:

   * Sum > 0 → move right.
   * Sum < 0 → move left.
   * Sum == 0:

     * Store the triplet.
     * Move left.
     * Skip duplicate left values.
7. Continue until the pointers meet.

---

## Code

```python
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        res = []
        nums.sort()

        for i, a in enumerate(nums):

            if i > 0 and a == nums[i - 1]:
                continue

            l, r = i + 1, len(nums) - 1

            while l < r:

                threeSum = a + nums[l] + nums[r]

                if threeSum > 0:
                    r -= 1

                elif threeSum < 0:
                    l += 1

                else:
                    res.append([a, nums[l], nums[r]])

                    l += 1

                    while l < r and nums[l] == nums[l - 1]:
                        l += 1

        return res
```

## Why Skip Duplicates?

Suppose:

```text
[-1,-1,-1,2]
```

If we don't skip duplicates:

```text
[-1,-1,2]
```

would be added multiple times.

Therefore,

```python
if i > 0 and nums[i] == nums[i-1]:
    continue
```

prevents duplicate first elements.

Similarly,

```python
while l < r and nums[l] == nums[l-1]:
    l += 1
```

prevents duplicate second elements.

This guarantees unique triplets.

## Complexity Analysis

### Time Complexity

Sorting:

```text
O(n log n)
```

Outer loop:

```text
O(n)
```

Inner Two Pointer traversal:

```text
O(n)
```

Overall:

```text
O(n²)
```

The `O(n²)` term dominates `O(n log n)`.

---

### Space Complexity

```text
O(1)
```

excluding the output list.

# Learnings

* Many `k-Sum` problems can be reduced to a smaller `k-1 Sum` problem.
* Sorting enables the Two Pointer technique.
* Duplicate handling is essential to avoid repeated triplets.
* Two Pointers reduce the complexity from `O(n³)` to `O(n²)`.
* 3Sum is built directly on the idea used in **Two Sum II**.
